# 桌面宠物 · PRD (Product Requirements Document)

> **项目名称**: MoePet (萌宠)
> **版本**: v0.1
> **目标用户**: 女朋友 (Mac 用户)，后续扩展 Windows
> **技术栈**: Tauri v2 + Rust + HTML/CSS/JS (Canvas 渲染)

---

## 1. 产品概述

一个运行在桌面上的二次元萌系宠物小程序。宠物可以在桌面上自由活动，拥有独立的状态系统，陪伴用户日常使用电脑。

### 核心价值
- **陪伴感**: 桌面上有一个活生生的萌宠
- **治愈感**: 可爱的画风和互动反馈
- **轻量无感**: 不占资源，不打扰用户

---

## 2. 功能需求

### P0 - 必须 (MVP)

| 功能 | 描述 |
|------|------|
| **桌面渲染** | 宠物窗口穿透桌面鼠标点击，背景透明，始终置顶 |
| **行走动画** | 宠物在桌面底部左右走动，有 idle/行走动画帧切换 |
| **状态系统** | 三种属性：心情、饱腹、精力，各自 0-100 随时间衰减 |
| **右键菜单** | 右键宠物出现菜单：喂食、玩耍、查看状态、退出 |
| **自动行为** | 根据状态值自动切换行为：散步(正常)、坐姿发呆(无聊)、睡觉(疲惫) |

### P1 - 重要

| 功能 | 描述 |
|------|------|
| **鼠标拖拽** | 左键按住可拖动宠物到桌面任意位置 |
| **抚摸反馈** | 鼠标在宠物身上划过时播放开心动画/表情变化 |
| **对话气泡** | 状态变化时弹出文字气泡（饿、困、开心、打招呼等） |
| **多套外观** | 至少 3 套换装/配色，可通过菜单切换 |

### P2 - 锦上添花

| 功能 | 描述 |
|------|------|
| **定时提醒** | 宠物会在设定的时间提醒喝水、休息 |
| **数据持久化** | 退出时保存状态，重启后恢复 |
| **屏幕边缘碰撞** | 走到屏幕边缘时转身或跳跃 |
| **小游戏** | 点击飞行的蝴蝶/光点等简单交互 |

## 3. 用户交互流程

```
开机启动 → 宠物出现在桌面底部右侧
    ↓
宠物自动行走（随机方向、速度）
    ↓
用户鼠标划过 → 宠物眨眼看鼠标 / 开心表情
用户拖拽 → 宠物跟随鼠标移动
用户右键 → 弹出功能菜单
    ↓
[喂食] → 饱腹 +30
[玩耍] → 心情 +20
[查看状态] → 弹出状态面板
[换装] → 切换外观
    ↓
空闲时自动行为切换:
  心情<30 → 垂头丧气走路
  饱腹<20 → 头顶冒饥饿图标，走向屏幕边缘"找吃的"
  精力<20 → 原地坐下打瞌睡
```

---

## 4. 状态系统设计

### 属性

| 属性 | 初始值 | 衰减速度 | 影响 |
|------|--------|---------|------|
| 心情 | 80 | 每 5 分钟 -1 | <30 消极动画 |
| 饱腹 | 80 | 每 10 分钟 -1 | <20 找食物行为 |
| 精力 | 100 | 每 3 分钟 -1 | <20 打瞌睡，<5 自动睡觉恢复 |

### 交互反馈

| 用户操作 | 心情 | 饱腹 | 精力 |
|---------|------|------|------|
| 喂食 | +0 | +30 | +0 |
| 玩耍 | +20 | -5 | -15 |
| 抚摸 | +10 | +0 | +0 |
| 睡觉(自动) | +5/分钟 | -1/分钟 | +10/分钟 |

## 5. 技术方案

### 架构

```
┌─────────────────────────────────┐
│         Tauri Shell (窗口)       │
│  ┌───────────────────────────┐  │
│  │   WebView (Canvas 渲染)    │  │
│  │   - 宠物精灵图渲染        │  │
│  │   - 动画帧控制            │  │
│  │   - 状态面板 UI           │  │
│  └───────────────────────────┘  │
│  ┌───────────────────────────┐  │
│  │   Rust Backend            │  │
│  │   - 状态管理与持久化      │  │
│  │   - 系统托盘              │  │
│  │   - 窗口穿透/置顶         │  │
│  └───────────────────────────┘  │
└─────────────────────────────────┘
```

### 关键实现

- **窗口穿透**: Tauri 的 `decorations: false` + `transparent: true` + 事件穿透
- **动画**: CSS Sprite Animation / Canvas 逐帧动画
- **状态管理**: Rust 端维护状态，通过 Tauri Events 推送到前端
- **持久化**: Rust 端 JSON 文件存储

### 宠物形象

前两版先用 **Canvas 绘制的矢量二次元角色**（圆脸大眼萌系），不依赖外部图片资源。后续可以替换为 Live2D 或逐帧精灵图。

## 6. 里程碑

| 阶段 | 内容 | 交付物 |
|------|------|--------|
| M1 原型 | 透明窗口 + 宠物绘制 + 行走动画 | 一个在桌面上走来走去的小人 |
| M2 MVP | 状态系统 + 右键菜单 + 喂食玩耍 | 可养成的桌面宠物 |
| M3 完善 | 对话气泡 + 拖拽 + 多外观 + 持久化 | 完整版 |

## 7. 非功能需求

| 项 | 要求 |
|----|------|
| 内存占用 | < 100MB |
| CPU 占用 | idle 时 < 1% |
| 启动时间 | < 2s |
| 打包体积 | < 20MB |
| 系统支持 | macOS 12+ / Windows 10+ |

---

## 8. 设计约束

- **不依赖外部图片/API**：宠物全部用 Canvas 绘制，开箱即用
- **不联网**：纯离线运行
- **守规矩**：不拦截系统事件、不弹窗打扰、不修改系统设置
